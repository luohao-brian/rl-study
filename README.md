# 强化学习迷宫求解器：从零理解 RL 核心概念

欢迎来到强化学习（Reinforcement Learning, RL）的入门实战项目！本项目是一个用于求解 5x5 迷宫的小型项目。我们希望通过这个直观的迷宫任务，帮助初学者不仅能跑通代码，还能真正理解强化学习背后的核心概念与算法原理。

项目中包含了两种经典的 RL 算法来求解同一个迷宫：
- **蒙特卡洛控制（MC, Monte Carlo Control）**
- **深度 Q 网络（DQN, Deep Q-Network）**

---

## 📖 核心概念 (Core Concepts)

要理解强化学习，我们首先需要搞清楚几个核心词汇。在我们的“迷宫求解”任务中，这些概念分别对应如下：

1. **环境 (Environment / Env)**
   - **定义**：智能体（Agent）所处的世界，它负责接收动作，并反馈新的状态和奖励。
   - **在本项目中**：就是一个 5x5 的网格迷宫。它包含了起点、终点和障碍物。对应代码中的 `MazeEnv` 类。

2. **状态 (State)**
   - **定义**：智能体在环境中的当前情况。
   - **在本项目中**：是智能体当前所在的坐标位置 `(row, col)`。

3. **动作 (Action)**
   - **定义**：智能体在某个状态下可以采取的行为。
   - **在本项目中**：智能体可以执行四个方向的移动：`上 (Up)`、`下 (Down)`、`左 (Left)`、`右 (Right)`。

4. **奖励 (Reward)**
   - **定义**：环境对智能体刚刚采取动作的反馈。正数表示鼓励，负数表示惩罚。
   - **在本项目中**：
     - 每走一步：`-1.0`（鼓励寻找最短路径）
     - 撞墙/非法动作：`-5.0`（惩罚撞墙）
     - 到达终点：`+10.0`（最终的胜利目标）

5. **策略 (Policy / $\pi$)**
   - **定义**：智能体的“大脑”或“行动指南”，决定在特定状态下应该采取什么动作。
   - **在本项目中**：我们的最终目标就是找出一个**最优策略**，让智能体在迷宫的任意一个格子里，都知道该往哪个方向走能最快到达终点。在代码中，策略通常表现为一个字典（如 `self.policy[state] = action`）或一个直接根据状态输出动作概率的神经网络。

6. **Q 值 (Q-Value / Action-Value) 与 Q字典**
   - **定义**：在某个状态 $S$ 下，采取动作 $A$ 后，未来预期能获得的**总奖励**。
   - **在本项目中**：无论是 MC 还是 DQN，本质上都是在努力估算最准确的 Q 值。只要知道了每个动作的 Q 值，我们在每个格子里只要挑选 Q 值最大的那个动作即可。在传统算法中，我们使用一个**Q字典（Q-Table）**，例如 `self.Q[(state, action)]`，它就像一个庞大的备忘录，记录了每走到一个特定格子并选择某个动作后，最终能拿到的平均总得分。

## 🧠 算法原理与核心代码逻辑

### 1. 蒙特卡洛方法 (MC, Monte Carlo Control)
**原理**：蒙特卡洛方法的核心思想是“**事后诸葛亮**”。它必须等智能体**走完完整的一局**（从起点直到终点，或者达到最大步数）后，再回过头来根据这一局的实际得分，更新途径的每一个状态-动作的 Q 值。
它就像一个玩游戏的人，每次必须玩到 Game Over 才能总结经验。

**核心逻辑与数据结构**：
- **数据结构 `Q-Table`**：在 `MCAgent` 中，我们维护了一个字典 `self.Q`。它就像一个巨大的查表，记录了迷宫中每个格子向四个方向走的预期得分。
- **轨迹采样**：在 `mc_train.py` 中，智能体不断在环境中尝试，生成一段段“经验轨迹”。轨迹结构为一系列的四元组 `(state, action, reward, next_state)`。
- **结合轨迹更新Q值**：在单局结束后，代码会**逆序遍历**这条轨迹。从最后一步倒推，累加获得的回报（Return/G）。每倒退一步，就用这个实际获得的累计回报 $G$ 去更新当前 `(state, action)` 在字典 `self.Q` 中的平均值。也就是说，整条轨迹就是为了拿到一个结局得分，然后把这个得分分摊到沿途走过的每个格子和动作上。
- **ε-贪婪策略 (Epsilon-Greedy)**：在训练时，智能体有 $1-\epsilon$ 的概率选择当前认为最优的动作，有 $\epsilon$ 的概率随机走（为了探索未知的格子）。

**On-Policy 与 Off-Policy (同策略与异策略)**：
- **On-Policy (同策略)**：学习和探索用的是**同一个**策略。比如本项目的 MC 方法，它用 ε-贪婪策略去迷宫里跑（探索环境生成轨迹），然后根据跑出来的结果，直接去更新和优化这个 ε-贪婪策略本身。因为是“边走边反思调整自己的当前策略”，所以是 On-Policy。
- **Off-Policy (异策略)**：学习和探索用的是**不同**的策略。比如本项目的 DQN 方法，它探索环境时是依靠 ε-贪婪策略乱走的（行为策略），但它学习更新 Q 网络时，总是假装自己在走下一步时一定会选择最优动作（目标策略）。它能够通过看“过去的自己”甚至“别人”的经验，超脱地学习出最优解，所以是 Off-Policy。

### 2. 深度 Q 网络 (DQN, Deep Q-Network)
**原理**：DQN 的核心思想是“**触类旁通**”与“**边走边学**”。它不再需要等完整的一局结束，而是每走一步就能学习。因为迷宫变大时，格子（状态）太多，传统的 Q-Table 装不下也学不过来，因此 DQN 使用**深度神经网络**来代替 Q-Table。

**核心逻辑与数据结构**：
- **神经网络 `QNetwork`**：代替了字典，输入是当前状态（坐标的独热编码），输出是 4 个动作的 Q 值。
- **经验回放 (Replay Buffer)**：核心数据结构！智能体每走一步，就把 `(state, action, reward, next_state, done)` 这一步的经验存入一个内存池。训练时，像抽盲盒一样从池子里**随机抽一批**经验来学习。这能打破数据之间的关联性，让神经网络学习更稳定。
- **目标网络与更新**：DQN 在计算“预期未来奖励”时，会结合环境给的即时奖励 `reward` 和神经网络对下一步状态 `next_state` 的估算。

## 🚀 运行环境与快速开始

### 依赖要求
- Python >= `3.12`
- 项目采用 `uv` 作为包管理工具，依赖 `torch`, `click`, `numpy`。

### 安装依赖
```bash
uv sync
```

### 运行代码
本项目提供了一个统一的命令行入口，方便大家对比测试不同的算法。

查看帮助：
```bash
uv run main.py --help
```

**运行蒙特卡洛 (MC) 求解**：
```bash
uv run main.py mc
```

**运行 DQN 求解**：
```bash
uv run main.py dqn
```

程序启动后，会首先打印出初始化的迷宫地图。随着训练的进行，你会看到训练日志。训练完成后，程序会打印出学习到的**最终策略棋盘**（箭头指示每个格子的最佳走向），并进行一次实际的策略验证。
